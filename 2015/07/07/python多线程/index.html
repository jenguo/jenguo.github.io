Python多线程的学习

标签（空格分隔）： Python

Python 标准库提供了 thread 和 threading 两个模块来对多线程进行支持。其中， thread 模块以低级、原始的方式来处理和控制线程，而 threading 模块通过对 thread 进行二次封装，提供了更方便的 api 来处理线程。 虽然使用 thread 没有 threading 来的方便，但它更灵活.

解释：

  1. 有一个全局变量，所有的线程都想修改它。
  2. 所有的线程应该在这个全局变量上加 1 。
  3. 有50个线程，最后这个数值应该变成50，但是它却没有。

为什么没有达到50？

  1. 在some_var是15的时候，线程t1读取了some_var，这个时刻cpu将控制权给了另一个线程t2。
  2. t2线程读到的some_var也是15
  3. t1和t2都把some_var加到16
  4. 当时我们期望的是t1 t2两个线程使some_var + 2变成17
  5. 在这里就有了资源竞争。
  6. 相同的情况也可能发生在其它的线程间，所以出现了最后的结果小于50的情况。
